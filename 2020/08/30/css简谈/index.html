BFC & IFC

BFC:(块级格式上下文)：形成独立区域，内部的样式设置不会影响外部的其他元素。

触发条件：

  • <html>元素
  • float: left;/float: right;
  • position: absolute;/position: fixed;
  • display: inline-block;
  • overflow: hidden;/overflow: scroll;/overflow: auto;

特点规则

  • 内部元素在垂直方向，从顶部开始一个接一个向下放置

  • 垂直方向距离由margin属性决定，属于同一BFC的相邻块级元素会发生外边距折叠

  • 可以包含浮动元素，用于清除内部浮动

  • 不会和float元素重叠

  • 经常用于实现两栏布局

  • 浮动元素参与高度计算

IFC(行级格式上下文)

触发条件：

块级元素中仅包含内联级别元素。

特点规则

  • 元素在一行中从左至右排列，当宽度不够时，将换行排列。
  • 每一个生成匿名行框（盒），包括该行所有行内框（盒）。
  • 水平方向上，当所有行内框（盒）的总宽度小于匿名行框（盒）宽度，水平排版由text-align决定。
  • 垂直方向上，行内框（盒）的对齐方式由vertical-align决定，默认为baseline即小写字母x最低点对应的线。
  • 在一行的所有元素会在该区域形成一个行框（盒），包含该行的最高点和最低点。
  • 元素排列的水平宽度，padding/border/margin所占用的长度会被计算在内。
  • 元素排列的垂直高度，由行框（盒）内实际高度最高的盒子计算而来，border,padding,margin不会计算在内。

属性float/定位position

属性float

常用float: left;/float: right;。

脱离文档流，容易造成父元素高度塌陷，需清除浮动。

设计初衷为实现文字环绕效果。

定位position

position: static;：默认，无定位，left，right，top，bottom设置无效。

position: absolute;：脱离文档流，相对最近的非static定位的祖先元素通过设置left，right，top，bottom等进行位置偏移。

position: relative;：相对自己的原始位置进行定位，通过设置left，right，top，bottom等进行位置偏移，原始位置依然占据文档流。

position: flxed;：脱离文档流，相对浏览器窗口通过设置left，right，top，bottom等进行位置偏移，不随页面滚动而变化。

position: sticky;：实现动态固定效果，在relative和fixed状态切换。

包含块&父元素

一个元素的尺寸和位置常常受到其包含块的影响，大多数情况下，包含块即为最近祖先块元素的内容区，但也并不总是这样。

确定元素包含块的过程完全依赖position属性：

如果position属性为static/relative/sticky，包含块一般为content-box。

如果position属性为absolute，包含块为距离他最近的非static祖先元素的padding-box。

如果position属性为fixed，包含块一般为视口viewpoint。

html5新特性及优缺点

新特性

  1. 语义化标签，增加<header>,<nav>,<footer>,<aside>,<section>,<article>等标签
  2. 增加多媒体标签<audio>,<video>用于替代flash
  3. Canvas绘图，利用JavaScript进行绘制
  4. 提供localStorage【没有时间限制】,sessionStorage【当浏览器窗口被关闭，数据被删除】本地存储
  5. 提供WebSocket全双工通讯协议

优点

  1. 统一标准，由W3C推荐
  2. 网页自适应设计，即时更新
  3. 多设备跨平台，可移植性强
  4. 提高了网页可用性和用户体验感
  5. audio,video等媒体标签，可替代flash
  6. 语义化标签，增强了可读性，同时在涉及网站索引和抓取时，对SEO【Search Engine Optimization 搜索优化，提高网站自然排名，增加曝光度，获取流量】友好

缺点

  1. 兼容性差，特性对于浏览器的支持程度不一，对IE9以下浏览器支持较差
  2. 部分浏览器殷引擎问题导致性能下降
  3. socket,storage可能造成安全问题，被盗取用户信息等

html语义化

  1. 用正确的标签做正确的事情
  2. html语义化标签使得页面结构更加清晰。便于搜索引擎解析及SEO
  3. 便于阅读理解和维护

常规计算

父元素600px，left条件：flex: 1 2 500px，right条件：flex: 2 1 400px，left/right实际长度为？

500 + 400 - 600 = 300 需要压缩
2 * 500 + 1 * 400 = 1400
left = 500 - 300 * 2 * 500 / 1400 = 285.72
right = 400 - 300 * 1 * 400 / 1400 = 314.28

父元素600px，left条件：flex: 1 2 300px，right条件：flex: 2 1 200px，left/right实际长度为？

600 - 300 - 200 = 100 需要放大
left = 300 + 100 * 1 / ( 1 + 2) = 333.33
right = 200 + 100 * 2 / ( 1 + 2) = 266.67

层叠上下文

文档中层叠上下文由满足以下任意一个条件的元素形成：

文档根元素<html>

position值为absolute或relative且z-index值不为auto的元素

flex容器的子元素，且z-index值不为auto

opacity属性值小于1的元素

transform/filter属性值不为none

层叠上下文可以包含在其他层叠上下文中，并且一起创建一个层叠上下文的层级

每个层叠上下文都可以完全独立于它的兄弟元素，当处理层叠时只考虑子元素

每个层叠上下文都是自包含的，当一个元素的内容发生层叠后，该元素将被作为整体在父级层叠上下文中按顺序进行层叠

层叠顺序

background/border < z-index < 0 < block box < float box < inline box < z-index = 0 < z-index > 0

在同一个层叠上下文中，如果元素具有明显的层叠水平标识（如z-index），层叠水平高的元素在上面，距离用户最近

当两个元素的层叠水平一致，层叠顺序相同时，DOM流后面的元素会覆盖前面的元素
